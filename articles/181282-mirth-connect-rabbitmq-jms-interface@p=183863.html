  • Mirth Connect RabbitMQ JMS Interface




    There has been some question in the past about adding a RabbitMQ connector to Mirth Connect. While, yes, this could be done, I had wondered if there was a way to interface with RabbitMQ using Connect as is. After doing some digging I found a RabbitMQ JMS client and set out test it. The following is how I tested and what the interface looks like using Connect.


    RabbitMQ Setup

    To avoid having to install and configure RabbitMQ myself, I used a RabbitMQ Docker image.

    To start the RabbitMQ container with name "some-rabbit", listening on port 5672, and serving the management interface on port 8081 run:

    Code:
    docker run -d --hostname my-rabbit --name some-rabbit -p 8081:15672 -p 5672:5672 rabbitmq:3-management
    Also, to enable extra plugins like Topic Exchange you can run the following command on the container:

    Code:
    docker exec -it some-rabbit rabbitmq-plugins enable rabbitmq_jms_topic_exchange
    To see a list of running plugins you can run the following command on the container:

    Code:
    docker exec -it some-rabbit rabbitmq-plugins list
    Management Interface

    With the container running you should be able to pull up the management interface at
    • http://localhost:8081
    • Username: guest
    • Password: guest
    Once logged in, create a queue by navigating to the "Queues" tab and create a queue named "myqueue". It should look something like this:




    Connect Setup

    Server

    Import the latest RabbitMQ libraries (you need both the JMS client and Java client) into a dedicated resource for use in your channels. Note, as of 4.0.1, there is a problem with using these libraries in JMS connectors so you will have to place them in custom-lib and set server.includecustomlib = true in mirth.properties. For example:
    • rabbitmq-jms-2.3.0.jar (https://mvnrepository.com/artifact/c...s/rabbitmq-jms)
    • amqp-client-5.12.0.jar (https://mvnrepository.com/artifact/c...mq/amqp-client)
    For documentation on the RabbitMQ JMS Client see:
    • https://www.rabbitmq.com/jms-client.html
    • https://github.com/rabbitmq/rabbitmq-jms-client
    • https://github.com/rabbitmq/rabbitmq-java-client
    JMS Sender



    ​
    Steps
    1. Set your channel dependencies to include the resource containing the RabbitMQ jars.
    2. Set your Destination Connector Type to: JMS Sender
    3. Use JNDI: No
    4. Load the ActiveMQ template
    5. Set Connection Factory Class to:
      1. com.rabbitmq.jms.admin.RMQConnectionFactory
    6. Set the brokerURL to:
      1. failover:(tcp://localhost:5672)?maxReconnectAttempts=0
    7. Set Username to: guest
    8. Set Password to: guest
    9. Set Destination Type to: Queue
    10. Set Destination Name to: myqueue
    11. Deploy channel
    12. Send a test message
    13. Observe the message is sent with no errors
    14. Observe in the RabbitMQ UI that "myqueue" has one message ready



    JMS Listener




    ​
    Steps
    1. Set your channel dependencies to include the resource containing the RabbitMQ jars.
    2. Set your Source Connector Type to: JMS Listener
    3. Use JNDI: No
    4. Load the ActiveMQ template
    5. Set Connection Factory Class to:
      1. com.rabbitmq.jms.admin.RMQConnectionFactory
    6. Set the brokerURL to:
      1. failover:(tcp://localhost:5672)?maxReconnectAttempts=0
    7. Set Username to: guest
    8. Set Password to: guest
    9. Set Destination Type to: Queue
    10. Set Destination Name to: myqueue
    11. Deploy channel
    Once that's done and you deploy your channel you should see a consumer registered in the RabbitMQ manager page. You should also see messages show up in the RabbitMQ listener that were sent by the RabbitMQ sender.
