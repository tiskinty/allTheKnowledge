                  Couple of things to look at...

                  1)On the source->tranformer (1). Make sure that the Message Segment and Mapping are not reversed. We have a 134 sites and have only used the Message Builder in one or two spots. I don't remember exactly where to look to verify the correct way. I do remember that we had it backwards. You would be sending the template message through with only a couple of fields from the original message instead of vice versa (See #2).

                  2)If there is no OBX.5.5, everything here will crash;

                  var seg = tmp['OBX']['OBX.5']['OBX.5.5'].toString(); //seg will = null
                  var PrinterName = seg.substring(0,seg.indexOf("~"));//can not call index of null
                  var temp = seg.substring(seg.indexOf("~")+1);//same as above making temp null
                  var TenetFileName = temp.substring(0,temp.indexOf("~"));//can not call index of null
                  var pdfData = temp.substring(temp.indexOf("~")+1);//same as above

                  channelMap.put("PrinterName", PrinterName);//setting null
                  channelMap.put("TenetFileName", TenetFileName);//setting null
                  channelMap.put("pdfData", pdfData);//setting null

                  a)if #1 is occuring there will be no OBX. Resolving this will solve this problem
                  b)if #1 is not occuring but logic around prevent null issues.
