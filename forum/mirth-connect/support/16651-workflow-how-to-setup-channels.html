              I believe these steps can be done in 1 channel, with some efforts.

              a) Create a Listener in Mirth that will receive HL7 from NCE. I am inclined to believe that the first message type, when (1)Nurse enters ID in NCE will be different when the (2)Nurse enters Patient ID. For the sake of discussion, lets assume we would be getting an MFN^M02, and ADT^A04, respectively.

              b) Extract the id in source transformer, and push it inside a channelMap variable. Also create a flag like nurseData=true for M02 message, and patientData=true for the A04 message.

              c) This is complicated step. Create your first Destination as a JS writer, and create a DB connection in it. Now, if the nurse flag is true, you run the SQL for Nurse, and when the patient flag is true, run the patient SQL, inside the same connection.

              d) Pass the result set to a function that generates an HL7 XML, and returns an XML object, that can be easily serialized into HL7. Push this XML object inside a channelMap.

              e) Add a new Destination, that has XML as inbound and HL7 as outbound data type. In the transformer just add this


              PHP Code:
              tmp=$(hl7XMLObject); 
              //this is the channelMap variable that has the XML object in it.

              The sender will send HL7 back to NCE server. Select your sender type.

              f) Now, comes the third HL7 message which consists of Demographics and BP results, extract information that you need in Source transformer, and pass it to channelMap.

              g) Add a DB writer destination, total 3 so far, which will insert data to a SQL server DB, which I think is not NCE per your description in point 3.

              h) Finally, for response you have to use Post processor script that shall check the status of all three destination, and returns an HL7 ACK in events all three are good, else a NACK. See around the forum for how to do that.
              ---

              I hope it helps.

                  If you want to have different channel/Listeners you will have problem tracking and connecting the dots, if a message fails. The other problem is that the response will be spread on three different channels, which would be a problem for you to send it back to NCE.
                  The efforts are more in the strategy I gave, but at least they all will be on single channel and time spent is less in tracking.

                  The points you mentioned in your post above, are very simple task. I can only advise you to search through the forum for relevant topic, or join Mirth Slack channel for real time assistance.
