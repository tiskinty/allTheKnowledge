    Mapping OBX Parametercode

    01-18-2023, 03:35 AM
    Hello Mirth and HL7 community,

    I have a project at the moment where I need to map the parameter names in the OBX segments with the LOINC code.
    I have now already tested a few variants, but have not found a solution for this, even when searching the forum I have not come across a solution.

    And we have the parameter names MDC, that means our OBX segments look like this:​

    Code:
    OBX|1|NM|150456^MDC_PULS_OXIM_SAT_O2^MDC|1.3.1.150456|98|262688^MDC_DIM_PERCENT^MDC|||||R|||20220915100744.0000+0100
    We would like to map these now, and I have thought of this in such a way that if in the element OBX.3.1 the number 150456 stands all necessary fields are to be adapted. So that the message looks like this:

    Code:
    OBX|1|NM|59408-5^Oxygen saturation in Arterial blood by Pulse oximetry^LN||98|%^^UCUM|||||F|||20201105094600
    Only the OBX segment for which these changes apply is to be mapped.

    Code:
    if (obx['OBX.3']['OBX.3.1'].toString() == 150456)
     {
       new msg['OBX']['OBX.3']['OBX.3.1'] = '59408-5';
       new msg['OBX']['OBX.3']['OBX.3.2'] = 'Oxygen saturation in Arterial blood by Pulse oximetry';
       new msg['OBX']['OBX.3']['OBX.3.'] = 'LN';

       new msg['OBX']['OBX.6']['OBX.6.1'] = '%';
       new msg['OBX']['OBX.6']['OBX.6.3'] = 'UCUM';
      }
    This is my current solution, but the corresponding fields are not changed.

    Furthermore I wanted to ask about a topic, which is probably easy to solve, but I still can't find a solution.
    In the HL7 message in the OBX segments I have to store the date or the time without seconds. Does anyone know a solution for this so that I reformat the existing data and specify only up to the minute?

    I thank you all in advance and hope that I have not picked up a topic that has already been mentioned.

    ​

    You'll want to traverse the list of OBXs.
    Code:
    for each (obx in msg.OBX) {
   if (obx['OBX.3']['OBX.3.1'].toString() == '150456') {
      obx['OBX.3']['OBX.3.1'] = '59408-5'
   }
}
    For date formatting you can work with DateUtil or new Date(). You can find examples of both on the forum.​​

            01-19-2023, 01:09 AM
            I found the solution to the above question and am posting it here so someone else doesn't make the same mistake I did:

            I had the WRONG date format. The HL7 date format is:
            Code:
            yyyyMMddHHmmss.SSSSZ
            .

            I have now solved it with the following code:​

            Code:
            msg['OBR']['OBR.7']['OBR.7.1'] = DateUtil.convertDate('yyyyMMddHHmmss.SSSSZ','yyyyMMddHHmm', msg['OBR']['OBR.7']['OBR.7.1'].toString());

                1. Always use null and empty checks before parsing.
                2. The first error is because one of the OBX.14 is empty.
                3. The second error is because one of the OBX.14.1 is having field repeats.
                4. You cannot delete a segment from the middle of the message and have it automatically reorganise. Deleting 6th OBX will not bump 7th to the 6th place.

                It would be helpful if we can see what your message looks like.
