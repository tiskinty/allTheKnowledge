    Build new XML message from Javascript reader source?

    02-10-2016, 05:39 AM
    I've been working on a javascript reader that takes in a pdf, strips out the text using iText and then assigns a few variables that I print out using logger.info.

    However, beyond this I'm not getting any message going through the channel. All I want to do is build a small XML message generated from the three variable values I have in the Javascript Reader.

    Source type is Delimited Text to XML ->

    Any ideas? I don't even get anything going into the destination channel writer. It processes the reader to the console and then I don't get anything going into the source transformer.

    I read this and tried to replicate in XML rather than HL7 but didn't have much joy:

    http://www.mirthproject.org/community/forums/showthread.php?t=14157


    I've pretty much got the data and things I want, I just need to able to access the javascript variables in the JScript Reader in the the transformer to fill in the XML template I have.


    UPDATED

    OK, I thought I'd post the updated status in the first post to help anyone else out and the things I learned whilst doing it. The aim of this channel was to read in a PDF file then strip out some information and build an XML message from it and forward onto a destination. After much trial and error and some help from the guys in this thread finally managed to get it working. So I'll detail a few things here - these steps are relevant to the below platforms:

    Mirth Connect 3.3.2
    vApp version on Linux

    Mirth does ship with a version of iText but there seems to be a discrepancy between the lowagie version and sourceforge version. I have included the file I have used in this post. I added this into my custom-libs.

    So I created a javascript reader and created the function to strip out the text from the PDF. Generic example below:


    PHP Code:
    function getText(src){
        var     fileIn = new Packages.com.itextpdf.text.pdf.PdfReader(src);
        var     fileExt;
        
        for (    i=1;i<=fileIn.getNumberOfPages();i++) {
                //logger.info(Packages.com.itextpdf.text.pdf.parser.PdfTextExtractor.getTextFromPage(fileIn, i));
                fileExt = Packages.com.itextpdf.text.pdf.parser.PdfTextExtractor.getTextFromPage(fileIn, i);
                logger.info(fileExt)
        }
            fileIn.close();
        return     fileExt
    } 
    The next part was to watch a directory for files with a *.pdf extension so I used an existing function from somewhere on this site. When I called the getText function I used a logger.info to see the formatting of the PDF - tables and different layouts are stripped out in different patterns. And from there I was able to use regex to pull out the data I needed. Note, this will only be really useful if all your PDF's are templated in the same fashion. The rest of the javascript contents are below:

    PHP Code:
    var directoryToMonitor = "<<YOUR_FOLDER_HERE>> e.g. /folders/inbox/";                                                //Generic javascript file reader script
    var fileDirectory = new Packages.java.io.File(directoryToMonitor);                                // Get the directoryList in that directory (returns File[])...
    var wildcardFileFilter = new Packages.org.apache.commons.io.filefilter.WildcardFileFilter("*.pdf");
    var     filenameFilter = new Packages.java.io.FilenameFilter(wildcardFileFilter);
    var     directoryList = fileDirectory.listFiles(filenameFilter);
    var     iNumFiles = directoryList.length;                                                        // Get the number of files in the directory...
    var retVal = new Packages.java.util.ArrayList();                                                // Use an ArrayList to return multiple messages...
    var i = 0;


    if (    iNumFiles > 0) 
        for (var     i = 0; i < iNumFiles; i++)
        {        
            var     file = directoryList[i];
            var     fileName = file.getName().toString().split("\\.");
                globalChannelMap.put("filename",fileName[0]);
                
            var     contents = getText(file);
                logger.info(contents);

            var     attID = contents.match(/(.*Hospital No.\s+)(.*)(\s+)/);                    //Regex match returns an array, index position [2] is the match group
                        if (!attID) 
                            {
                                    logger.info("NOT FOUND");
                                var     att = "UNKNOWN";
                            }
                    else     {
                                    logger.info("Attendance ID is " +attID[2]);
                                var     att = attID[2];
                            }


            var     nhsID = contents.match(/\bNHS No.\s*(\d{10})/);                        //Regex match returns an array, index position [2] is the match group
                        if (!nhsID) 
                            {
                                    logger.info("NOT FOUND");
                                var     nhs = "UNKNOWN";
                            }
                    else     {
                                    logger.info("NHS No is " +nhsID[1]);
                                var     nhs = nhsID[1];
                            }

            var     pCode = contents.match(/[^,]*(?=[\n\r]+DOCTOR)\b/);
                    if (!    pCode) 
                            {
                                    logger.info(pCode);
                                    logger.info("NOT FOUND");
                                var     pcode = "UNKNOWN";
                            }
                    else     {
                                    logger.info("Postcode is " +pCode[0]);
                                var     pcode = pCode[0];
                            }

                    if(!    Packages.org.apache.commons.io.FileUtils.deleteQuietly(file))    // Delete the file now that is has been read...
                                {
                                    logger.info("Could not delete: " + theFile.getAbsolutePath()); 
                            }


            globalChannelMap.put("att",att);
            globalChannelMap.put("nhs", nhs);
            globalChannelMap.put("pcode", pcode);

        return     true;
        } 

    Things of note:

    1. Putting 'return true;' outside the main loop causes the file reader loop to not work correctly. It seems to get confused with what it needs to process and in what order leading to it generating lots of null messages;
    2. Initially my problem was with passing through variables in a channel map. In a javascript reader channelMap doesn't work and you need to use globalChannelMap.put for it to pass through to your transformers.
    3. The text returned is an array which is why the variables have an index position. Index position [0] is usually the entire group of text. You might need to play with indexes depending on the length of your string.
    4. logger.info(contents) will log the outputted text from the PDF into the console. You can then copy this out and use something like https://regex101.com/ to build your regex expressions.

    Then you need to create a sample XML message template. In this case I used something very simple like this:

    PHP Code:
    <eDischarge>
        <    msg_attendance_id></msg_attendance_id>
        <    msg_nhs_id></msg_nhs_id>
        <    msg_pcode_id></msg_pcode_id>
    </    eDischarge> 
    Then on the javascript reader source, edit transformer and use a transformer to pass the variables into the XML template like this:

    PHP Code:
    var att = globalChannelMap.get("att");
    var     nhs = globalChannelMap.get("nhs");
    var     pcode = globalChannelMap.get("pcode");

    channelMap.put("filename",globalChannelMap.get("filename"));

    tmp['msg_attendance_id'] = att;
    tmp['msg_nhs_id'] = nhs;
    tmp['msg_pcode_id'] = pcode; 
    I have included a sample channel, a sample PDF (this doesn't work properly as when editing it, it didn't save in the same format, but you can get an idea how it works from it) and the iText jar I used to get this working. Thanks to jack for the globalChannelMap pointer. Bit obscure that. Hope it helps someone in the future. There's some of the tidy up work like the trims I've not included as they will be case specific.
