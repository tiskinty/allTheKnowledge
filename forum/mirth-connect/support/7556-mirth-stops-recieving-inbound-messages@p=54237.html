    Mirth Stops recieving inbound messages

    07-24-2012, 11:07 AM
    Hi,

    I have a setup of Mirth with VA VistA for HL7 inbound messages processing to HIS. Version we are using is 1.8.2. This inbound channel is created as an LLP listener and the transforms and filters the messages and inserts into SQL Server 2005 DB. For past one month we are facing issues where the Mirth stops receiving inbound messages in this channels and the Filers in VistA goes into hang state. After a detail analysis we found that the messages in VistA are in "Awaiting Confirmation" status. The Send ACK check box in the channel is set to "yes" but the AA,AE,AR are all set to "No". Setup screenshot attached.

    For resolving this issue, we need to always do a deploy all for channels or sometime Mirth services are restarted. But since this is a temporary work around, I need why this problem is arising.

    Also, since last month we are experiencing high CPU utilization. Mirth is installed on Windows 2008 server.

    Any help regarding these issues will be great.

    Regards,
    Saurabh
