    In the Channels screen, on the left you will see "Edit Code Templates." Go there, and create a new Code Template Library and associate it with any channels that need the function. Then create a new Code Template in your Library and paste "The Code" from the forum post you linked.

            Because milliseconds doesn't exist in the original format, I'd expect you'd need to create/append something that wouldn't be accurate. Since MSH-7 represents the time of the event, you may consider pulling in the current date/time formatted with milliseconds. Another option might be to check the EVN segment for a date/time that's consistent with MSH-7 but has milliseconds, then map it to MSH-7. Otherwise, I'd think the upstream system would need to send the millisecond format.

            Current date/time w/milliseconds:
            Code:
            msg.MSH['MSH.7'] = DateUtil.getCurrentDate('yyyyMMddHHmmssSSS')

                06-09-2020, 07:19 PM
                To get the current date/time in Pacific time, you can do
                Code:
                with (JavaImporter(java.time.ZonedDateTime, java.time.ZoneId, java.time.format.DateTimeFormatter)) {
    msg.MSH['MSH.7'] = ZonedDateTime.now(ZoneId.of('America/Los_Angeles')).format(DateTimeFormatter.ofPattern('yyyyMMddHHmmssSSS'));
}
                This should respect daylight savings, too, if your Java is up to date.
