      Instead of accepting HL7, accept XML instead, in which the client is expected to send the user ID, password, and the actual HL7 message. Then you can do your validation in the source filter and place a response object in the response map (an ACK for success or a NACK for failure, etc.). Then choose that custom variable from the "Respond from" combo box on the source connector settings.

                  Check the attached channel.
                  Code:
                  var hl7=Packages.java.lang.String(FileUtil.decode(msg['hl7'].toString()));
var xml=SerializerFactory.getHL7Serializer().toXML(hl7);
channelMap.put("hl7message",new XML(xml));
                  The above code will decode your msg['hl7'].toString() and creates a valid HL7 message object accepted by mirth and stores it in channel map variable hl7message which you are using at destination transformer.

                      return msg.username.toString() == 'mdshl7' && msg.password.toString() == 'h$los2ws';

                      Code:
                      MSH|^~\&

                      tmp = SerializerFactory.getHL7Serializer().toXML(new java.lang.String(org.apache.commons.codec.binary.Base64.decodeBase64(msg.hl7.toString()),'ISO-8859-1'));
