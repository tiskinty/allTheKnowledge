    Yep, you're on the right track. Basically if your Database Reader is querying just one column, the inbound message for your channel will look something like this:

    Code:
    <result>
	<OCS>OCS|REPORT|123456|7891011||</OCS>
</result>
    From there, you can use a transformer to extract the data you need and perform queries. Here's an example:

    Database Reader SQL
    Code:
    SELECT OCS FROM ocstest WHERE processed = false;
    Source Transformer (XML->HL7v2.x)
    • Outbound Template: "MSH|^~\&"
    • Code:
      Code:
      tmp = new XML(SerializerFactory.getHL7Serializer().toXML('MSH|^~\\&\r'+msg.ocs.toString()+'\r'));


    Destination Transformer (HL7v2.x->XML)
    • Outbound Template:
      Code:
      <result>
	<action/>
	<patient>
		<firstname/>
		<lastname/>
	</patient>
	<order>
		<value/>
		<units/>
	</order>
</result>
    • Code:
      Code:
      var dbConn;

// General, query-independent info goes here
tmp.action = msg.OCS['OCS.1']['OCS.1.1'].toString();


try {
	// Create the database connection
	dbConn = DatabaseConnectionFactory.createDatabaseConnection('org.postgresql.Driver','jdbc:postgresql://localhost:5432/test','user','pass').getConnection();

	// Query for patient information
	var ps = dbConn.prepareStatement('SELECT name_first, name_last FROM patient WHERE id = ?');
	ps.setString(1,msg.OCS['OCS.2']['OCS.2.1'].toString());
	var rs = ps.executeQuery();

	if (rs.next()) {
		// Set the patient information
		tmp.patient.firstname = rs.getString(1);
		tmp.patient.lastname = rs.getString(2);
	}

	// Query for order information
	ps = dbConn.prepareStatement('SELECT value, units FROM orders WHERE id = ?');
	ps.setString(1,msg.OCS['OCS.3']['OCS.3.1'].toString());
	rs = ps.executeQuery();

	if (rs.next()) {
		// Set the order information
		tmp.order.value = rs.getString(1);
		tmp.order.units = rs.getString(2);
	}
} finally {
	// Always close the connection
	try {dbConn.close()} catch(e) {}
}


    Database Reader On-Update SQL
    Code:
    UPDATE ocstest SET processed = true WHERE ocs = ${ocs}


    So, if you had a row in ocstest (ocs:"OCS|REPORT|123456|7891011||", processed:FALSE), a row in patient (id:"123456", name_last:"Doe", name_first:"Jane"), and a row in orders (id:"7891011", value:"42", units:"mg"), then you would have the following inbound message:

    Code:
    <result>
    <ocs>OCS|REPORT|123456|7891011||</ocs>
</result>
    And the resulting outbound message:

    Code:
    <result>
    <action>REPORT</action>
    <patient>
        <firstname>Jane</firstname>
        <lastname>Doe</lastname>
    </patient>
    <order>
        <value>42</value>
        <units>mg</units>
    </order>
</result>
    Instead of compiling an aggregated XML file at the end, what many people do is actually create an HL7 message. Using the above example, you could easily create an ORU message for example.
