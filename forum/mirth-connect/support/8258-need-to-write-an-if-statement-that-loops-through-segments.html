    Need to write an If Statement that loops through segments

    12-03-2012, 05:11 PM
    I have a situation on a pharmacy order where sometimes there is no RXC segment, sometimes there IS a single RXC segment, & sometimes, there are multiple RXC segments, but always there is an RXE segment When there is no RXC segments, the dose information is present in RXE 4 & 5. When there is an RXC segment the dose information is present in the RXC segment, but not in the RXE segment - BUT the RXE segment will mostly mirror the RXC segment. In the case where there are multiple RXC segments, the main medication is in one of the RXC segments but other RXC segments will contain other information such as additive items - like saline.

    In all instances where there is an RXC segment, the RXE segment RXE-2 will match one of the RXC-2 segments.

    My receiving vendor wants all of the dosing information to be populated into the RXE segment for the main medication even if it is in the RXC segments, so I need to write an if statement that will look at ALL of the RXC segments and find the one that matches the RXE-2 segment. Then take the dose and amount date from RXC-3 & RXC-5 segments & put them into RXE 4 & 5.

    I could easily do this with my other interface engine, but I am trying to learn how to do this in Mirth. Still being new to javascript, I'm fairly comfortable on making the If statement, but don't know how to loop through each instance of the RXC segments to do a match. Essentially, here is what I think I need for my statement:

    For Each instance of RXC
    if (msg RXC2.1 == RXE2.1) {
    RXC3 = RXE4;
    RXC5 = RXE5;
    }

    How do I write this in javascript?

    Thanks

    so I need to write an if statement that will look at ALL of the RXC segments and find the one that matches the RXE-2 segment
    You can do sometime like this. Just fine tune the code as per your requirements.
    Code:
    for each(seg in msg..RXC)
{
if(seg['RXC.2']['RXC.2.1'].toString()==msg['RXE']['RXE.2']['RXE.2.1'].toString())
{
msg['RXE']['RXE.4']['RXE.4.1']=msg['RXC']['RXC.3']['RXC.3.1'].toString();
msg['RXE']['RXE.5']['RXE.5.1']=msg['RXC']['RXC.4']['RXC.4.1'].toString();
}
}
    The above code will iterate over all RXC segment and checks whether RXC 2.1 matches RXE 2.1. If it founds any match then it will store RXC.3.1(Component Amount) in RXE.4.1(Given Amount- Maximum) and RXC.4.1(Component Units) in RXE.5.1(Given Units)

                            That worked but it didn't bring in every instance of the FA AL1 segments, only the last one. How do I get it to bring in all of them.
                            It will store only the last al1 segment with "FA" because we have created only one AL1 segment. You have to change your logic according to suffice your requirements. You can either create multiple AL1 segment or remove other AL1 segment except the AL1 containing "FA"
                            Code:
                            var i=1;
for each (al1 in msg.AL1)
{
    if (al1['AL1.2']['AL1.2.1'].toString() != "FA")
    {
           delete msg.children()[al1.childIndex()];
    }
    else
    {
           al1['AL1.1']['AL1.1.1']=i;
           i++;
    }
}
