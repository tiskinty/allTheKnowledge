    Hello melonman,

    It would be helpful if you posted a sample of the data that you are trying to process, with any PHI removed.

    You can iterate over the rows in the delimited text by using the code template "Iterate Over Segment" in a Javascript transformer step. This code template looks like:

    Code:
    for each (seg in msg..SEG) {
	var sample_value = seg['SEG.1']['SEG.1.1'].toString();
}
    If your data looked like this:
    Scheller,Brian
    Doe,John

    You would want to modify it to be something like:

    Code:
    for each (seg in msg..row) {
	var fname = seg['column1'].toString();
        if (fname == "Brian") {
             seg['column2'] = "NewLastName";
        }
}
    Looking at your code sample, you are attempting to retrieve data from the incoming message, but I do not see anywhere you are attempting to update a value in the incoming message.

                        Before you look at anything, you can see a glaring problem between your Message template and message tree. in your template you have what should be 3 lines, but in your tree you only see one row.

                        After that, change your line delimiter to \r and uncheck "Ignore carriage returns" in your message properties and look at the new message tree. You should now see 3 lines.

                        Another useful tool is the logger. alter your code in your transformer to the following:

                        Code:
                        for each (seg in msg..row) {
		counter++;
		column = columnStart + counter + columnEnd;
		var fname = seg['column1'].toString();
		logger.info(fname);
                logger.info(counter);
			if (fname == "1" && counter == 22) {
					seg['column23'] = " ";
			}
	}
                        so you can see what is happening in your process. You will never hit the exclusion of fname in your current processing as it will never equal "1" it looks like.

                        Finally, the batch process should actually be set to "No" to achieve the counting you are looking for.
